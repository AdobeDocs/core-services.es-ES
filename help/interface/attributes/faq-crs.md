# Preguntas más frecuentes, limitaciones y prácticas recomendadas

Preguntas más frecuentes y prácticas recomendadas para Atributos del cliente en Analytics y Destinatario.

## Prácticas recomendadas y limitaciones  {#section_7F5189B3DAA84EE6865B91D2026EE05A}

Directrices y limitaciones al utilizar atributos [!UICONTROL del]cliente.

| Problema | Descripción |
|--- |--- |
| [!UICONTROL Limitaciones de suscripción de atributos] del cliente | Al actualizar a Analytics Premium, se produce un retraso de 24 horas antes de que estén disponibles atributos adicionales. Durante este periodo, puede aparecer el error Máximas suscripciones a atributos. |
| Varios inicios de sesión en el mismo dispositivo | Al utilizar Atributos del cliente para cargar perfiles de cliente en un origen de datos, Adobe recomienda que no se compartan los mismos usuarios (es decir, el mismo ID de Experience Cloud). Si lo hace, el servicio ECID, que persiste en el dispositivo, puede vincular varios usuarios con el mismo ID de Experience Cloud, lo que provocará resultados inesperados en [!DNL Target]. **Nota:** En el caso de Mobile, el ECID es permanente después de instalar la aplicación Mobile y debe volver a instalar la aplicación para generar un nuevo ECID. Para Web, se genera un nuevo ECID después de borrar la cookie del explorador. |
| Límite diario de carga de frecuencia | Adobe recomienda actualizar los atributos del cliente solo una vez al día. Debe esperar al menos 24 horas para cargar otro archivo de datos de perfil del cliente para el mismo conjunto de perfiles. |
| ID de Analytics personalizado (`s.visitorID`) | El establecimiento de un ID de cliente mediante `s.visitorID` es un método para identificar a los usuarios en Analytics. Sin embargo, las integraciones en las que se exportan o importan datos de Analytics mediante el servicio de ID no funcionarán cuando se identifique un visitante mediante `s.visitorID.`<br>Esto incluye, entre otras cosas, Audiencias compartidas, Analytics para Adobe Destinatario (A4T) y Atributos del cliente.<br>Estas integraciones no admiten la configuración de un ID de Analytics personalizado. |
| Limitaciones de longitud de caracteres en Analytics | Al crear una suscripción de Analytics, las longitudes de campo de los archivos cargados se truncan a 255. |

## FAQ about Customer Attributes {#section_E47866EEA83348E09FE43CEC5E44C461}

| Pregunta | Respuesta |
|--- |--- |
| ¿Puedo recibir notificaciones sobre el estado de carga de Atributos del cliente? | Sí. Consulte [Administración de notificaciones](https://docs.adobe.com/content/help/en/core-services/interface/manage-users-and-products/organizations.html#concept_0105453AD71847B8BFCAF4A40915F157). |
| ¿Qué debo hacer para empezar a usar Atributos del cliente? | <ol><li>Obtenga el aprovisionamiento. Si es cliente de Analytics, Adobe le proporcionará Atributos del cliente. Si solo usa Adobe Destinatario y no tiene Analytics, debe solicitar el aprovisionamiento de servicios principales poniéndose en contacto con el Servicio de atención al cliente.</li> <li>Póngase en contacto con su equipo de CRM. Descubra qué tipo de datos de clientes están disponibles y que resultarían interesantes para usar en Analytics y en Experience Cloud.</li><li>Implementar servicios principales. Consulte [Activación de las soluciones para servicios](https://docs.adobe.com/content/help/es-ES/core-services/interface/about-core-services/core-services.html) principales para ver los pasos para modernizar la implementación. (Consulte la sección sobre la sincronización de ID de cliente para obtener información importante).</li></ol> **Nota:** Hay disponible una sección de preguntas más frecuentes para implementar los servicios principales  [ aquí](https://docs.adobe.com/content/help/en/core-services/interface/manage-users-and-products/faq.html#concept_13219B4E51784577B6FF78AAA203DE91). |
| ¿Cuántos atributos de cliente puedo utilizar? | You can upload hundreds of `.csv` columns to the customer attribute service. Sin embargo, al configurar suscripciones y seleccionar atributos, se aplican los siguientes límites (por grupo de informes), según las soluciones que posea:  <ul><li>Foundation: 0</li><li>Select: 3</li><li>Prime: 15</li><li>Ultimate: 200</li><li>Standard: 3 en total</li><li>Premium: 200</li><li>Adobe Destinatario Standard: 5</li><li>Adobe Destinatario Premium: 200</li></ul> |
| ¿Se requiere la migración al servicio Experience Cloud ID? | La migración depende de las soluciones que utilice. <ul><li>Adobe Analytics:  Se recomienda encarecidamente </li><li>Adobe Destinatario: Requerido. </li></ul><br> El uso del servicio de ID mejora la funcionalidad y abre las puertas al uso de la funcionalidad más reciente de Experience Cloud, incluidas las audiencias en tiempo real, la modernización de Adobe Destinatario, la integración de Analytics y el seguimiento de Video Heartbeat. <br> Para obtener más información, consulte [Activación de soluciones para servicios](https://docs.adobe.com/content/help/es-ES/core-services/interface/about-core-services/core-services.html)principales. <br>** Nota:** El servicio [de ID de](https://docs.adobe.com/content/help/es-ES/id-service/using/intro/overview.html) Experience Cloud es la implementación modernizada de lo que antes se conocía como servicio de ID de visitante de _Analytics._ |
| ¿Cómo se relaciona la funcionalidad de atributos del cliente con Adobe Audiencia Manager? | Aunque el Administrador de Audiencias puede recibir datos para realizar la identificación de audiencias, no puede realizar funciones de análisis que vinculen atributos a datos de comportamiento históricos ni proporcionar las funciones de sistema de informes, análisis y segmentación disponibles en Adobe Analytics. [!UICONTROL Personas] permite que los datos enriquecidos de todas las soluciones se unan y asocien con un solo ID para su uso en Experience Cloud. <br>En Adobe Destinatario, los atributos del cliente aparecen como atributos individuales que se pueden combinar con otras reglas para crear audiencias. Las Audiencias compartidas con el servicio [!UICONTROL Personas] son audiencias completas que no se pueden modificar. |
| **(Solo Analytics)** ¿En qué se diferencia esta funcionalidad de la que se proporciona en Analytics Premium? | En el pasado, los clientes interesados en combinar datos de atributos del cliente con datos de Analytics dependían en gran medida de la herramienta del área de trabajo de datos para esta funcionalidad. Los atributos del cliente exponen esta funcionalidad a una audiencia más amplia al proporcionar Atributos del cliente como dimensiones y métricas en informes y análisis, análisis ad hoc y creador de informes. Los clientes de Analytics Standard tendrán acceso a Atributos del cliente, pero con capacidades limitadas. Los clientes de Analytics Premium pueden disfrutar de todas las funciones. |
| **(Solo Adobe Destinatario)** ¿Puedo cargar o precargar datos para clientes que Adobe Destinatario nunca ha visto? | Sí. Cuando el visitante realiza su primera solicitud a Adobe Destinatario, el sistema recopila la información existente que tenemos sobre ellos de Atributos del cliente y utiliza esos datos para el establecimiento de objetivos. **Nota:**  La recuperación de estos datos puede tardar hasta 20 minutos desde la primera interacción del visitante con Adobe Destinatario. |
| **(Solo Adobe Destinatario)** ¿Puedo crear una superaudiencia combinando datos de atributos del cliente con datos de audiencia compartidos? | No. Los datos de audiencia compartida son una audiencia completada. |
| **(Solo Adobe Destinatario)** ¿En qué se diferencia la funcionalidad Atributos del cliente de la API de perfil masivo de Adobe Destinatario? | La API de perfil masivo permite que los perfiles de Adobe Destinatario se actualicen directamente a través de la API, ya sea para un perfil individual o de forma masiva. La capacidad es similar a Atributos del cliente, con las siguientes diferencias clave:<ul><li>La API de perfil es una llamada a la API de REST y Atributos del cliente utiliza FTP.</li><li>La API de perfil de Adobe Destinatario solo envía datos a Adobe Destinatario en lugar de a todo Experience Cloud.</li><li>Los atributos del cliente proporcionan una interfaz sencilla para crear y administrar estos datos externos.</li></ul> |
| **(Solo Adobe Destinatario)** ¿La carga de datos desde Atributos del cliente a Adobe Destinatario amplía la duración del perfil del visitante de Adobe Destinatario? | Sí. Consulte [Duración del perfil del visitante](https://docs.adobe.com/content/help/en/target/using/audiences/visitor-profiles/visitor-profile.html) en la ayuda de Adobe Target. |
| **(Solo Adobe Destinatario)** ¿Puedo realizar destinatarios en los datos cargados en Atributos del cliente inmediatamente después de que el ID del cliente identifique el visitante? | Sí. En la llamada al servidor a Adobe Destinatario, que incluye el ID de terceros de mbox, estarán disponibles todos los datos de atributos del cliente. |
| **(Solo Adobe Destinatario)** ¿Qué representa la columna Estado **[!UICONTROL de]** sincronización para los archivos cargados en el origen de atributos del cliente? | Para ver el número de registros publicados y sincronizados por Adobe Destinatario, haga clic en el icono Estado de sincronización con un archivo de atributos específico. `Sync %` es una métrica en tiempo real que especifica el % de perfiles que se han sincronizado en Adobe Destinatario.<br> **Nota:** Los atributos pueden tardar hasta 24 horas en sincronizarse con Adobe Destinatario. |
| ¿Qué representan las métricas de carga de archivos en la fuente de atributos del cliente? | Puede comprobar el estado de los atributos cargados en Atributos del cliente con la ayuda de las siguientes métricas: <ul><li>Registros:  Número de registros en el archivo de atributos.</li><li>**Nuevos registros:** Número de registros nuevos presentes en el archivo de atributos.</li> <li>**Registros actualizados:** Número de registros que ya existen en Atributos del cliente con valores actualizados en el archivo.</li><li>**Todos los datos (registros):** Número total de registros cargados correctamente en Atributos del cliente.</li></ul> |
